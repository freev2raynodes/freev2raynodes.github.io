第一章 卸载前的战略准备

1.1 绘制"V2Ray生态地图"

资深用户可能在不同位置部署过多个组件：
- 核心程序：通常位于%ProgramFiles%\V2Ray或/usr/local/bin/v2ray
- GUI客户端：如Windows的V2RayN、macOS的Qv2ray
- 配置文件：分散在/etc/v2ray/、AppData\Roaming等隐蔽目录
- 浏览器插件：部分用户会配合SwitchyOmega等扩展使用

建议通过whereis v2ray（Linux/macOS）或全盘搜索v2ray.exe（Windows）建立完整清单。

1.2 备份的艺术

使用tar -czvf v2ray_backup.tar.gz /etc/v2ray或7-Zip打包相关目录，建议同时保存MD5校验值。

FAQ

Shadowrocket 在使用订阅时要注意什么？
在 Shadowrocket 中添加订阅时，应确保链接以 https 开头且格式正确。此外，若节点更新失败，可尝试关闭缓存或重新获取订阅链接，以保证配置文件同步最新节点数据。
VMess 节点的 alterId 参数还需要设置吗？
在新版 V2Ray/Xray 内核中，alterId 已被废弃，不再影响认证逻辑。建议用户设置为 0 或删除该字段，以避免连接异常或不兼容情况。
如何防止 Clash 被系统防火墙阻断？
请在防火墙设置中允许 Clash 的执行文件通过网络访问，并开放其监听端口。若使用 Windows Defender，可在防火墙高级规则中手动添加入站和出站规则，确保代理通信正常。
Matsuri 的延迟测试准确吗？
Matsuri 的延迟测试使用 ICMP 与 TCP 双重检测方式，可较准确反映节点实际响应时间。不过受网络波动影响，建议多次测试并结合速度测试结果综合判断。
Clash 的 Profile URL 是否可以同时添加多个？
可以。Clash 支持导入多个 Profile URL，用户可在“配置”面板切换不同订阅源。多个配置之间互不干扰，方便区分不同地区或用途的线路。
如何把 Clash 规则与本地 hosts 文件结合使用？
可在 Clash 的 rules 中优先设置域名直连规则，并在系统 hosts 中添加常用站点的固定解析以加速 DNS 解析。注意 hosts 优先级高于 DNS，因此修改 hosts 需谨慎以避免与真实解析冲突。
Matsuri 节点延迟排序功能如何使用？
Matsuri 会自动测试节点延迟，并根据结果排序。用户可以选择延迟最低的节点使用，从而优化访问速度和稳定性，尤其适合游戏、视频和海外网站加速。
V2Ray 的 QUIC 传输模式有哪些优势？
QUIC 传输减少握手次数，支持多路复用和 UDP 基础，降低延迟并提高吞吐量。在不稳定或高丢包环境下，QUIC 可保证视频、游戏或即时通讯的稳定性和流畅体验。
WinXray 的 PAC 模式适合哪些使用场景？
PAC 模式可根据规则文件动态判断流量是否走代理。适合访问国内外混合网站环境，无需手动切换节点，程序自动分流，提高网络访问效率和稳定性。
Mihomo 是否比 Clash Meta 更快？
Mihomo 是 Clash Meta 的优化分支，采用最新 Go 语言版本编译并针对多线程性能进行了改进。整体运行效率略高，特别是在多节点切换和 DNS 分流方面表现更流畅。
